5데이터 흐름 — 부팅부터 저장까지

1
로그인
Auth.login() → Apps Script ?action=auth → 48자 토큰 발급 → localStorage[tom_token] 저장. 비밀번호는 서버(Apps Script) 코드 안에만 존재.
2
초기 데이터 일괄 로드
DataLoader.loadAll() → ?action=getAllData 1회 호출로 lots·daily·invoices·customers·targets·shipments·settings 모두 수신 → 각각 normalize 함수로 타입/날짜 정리 → Store.set*() 저장.
3
페이지 진입
Nav.go('dash') → 모든 .pg 숨기고 대상만 표시 → Pages.X.render()가 Store.get*()로 메모리에서 데이터를 읽어 DOM 렌더.
4
사용자 입력 (낙관적 업데이트)
페이지가 Store.upsertLot/Daily/...로 메모리 즉시 갱신 → Api.append/update/delete가 큐에 push → 300ms 디바운스 후 순차 flush. UI는 서버 응답을 기다리지 않음. 실패 시 1회 재시도, 그래도 실패하면 토스트로 알림.
5
자동 변경 로그
Api.log(category, action, lotNo, summary) 호출로 logs 시트에 한 행 기록. 업데이트 현황 → 데이터 변경 로그 탭에서 조회.
6
크롤러 (별도 트랙)
GitHub Actions cron(매일 KST 11:00) → crawler/crawl_dram_price.py가 Playwright로 TrendForce 페이지 4개 테이블 파싱 → gspread로 DRAM Price 시트북에 누적 append. 마지막에 .last_crawl 커밋해 스케줄 자동 중지를 방지.
7
배포
main 브랜치에 푸시되면 .github/workflows/deploy.yml이 GitHub Pages로 정적 자산 그대로 업로드. 별도 빌드 단계 없음.

6운영 — 보안 · 자동화 · 외부 의존

🔒보안 모델

  • 비밀번호 Apps Script PASSWORD 상수에만 존재 — 프론트 코드와 GitHub에는 노출 안 됨
  • 토큰 48자 랜덤, PropertiesService에 JSON 배열로 저장, 8시간 만료, 슬라이딩 갱신(잔여 50%↓ 시)
  • 다중 기기 최대 50개 토큰 동시 보유 (기기/탭별 독립), 초과 시 가장 오래된 것부터 제거
  • 잠금 5회 실패 시 3분간 로그인 차단
  • 동시성 토큰 저장은 LockService로 보호
  • 전송 로그인은 POST(비밀번호) / 데이터는 토큰을 URL 또는 body에 포함

⚙자동화 (GitHub Actions)

  • deploy.yml main 푸시 트리거 → GitHub Pages 배포 (configure-pages → upload-pages-artifact → deploy-pages)
  • crawl_dram_price.yml 매일 02:00 UTC (KST 11:00) → Python 3.11 + Playwright Chromium → 시트 append → .last_crawl 자동 커밋(스케줄 비활성화 방지)
  • 시크릿 DRAM_PRICE_SHEET_ID, GCP_CREDENTIALS (서비스 계정 JSON)

⬡외부 라이브러리

  • Chart.js 4.4.1 + datalabels 플러그인 — 모든 차트
  • xlsx-js-style 1.2.0 — 엑셀 다운로드 (셀 스타일 포함)
  • Pretendard + DM Mono — 폰트 (jsDelivr/Google Fonts CDN)
  • Playwright + gspread — 크롤러용 (런타임에만 GitHub Actions에서 사용)

↻저장 큐 동작

  • 일반 저장 Api.append/update/delete → 300ms 디바운스 → 순차 flush, 실패 1회 재시도, 즉시 {success:true, optimistic:true} 반환
  • 중요 저장 Api.appendNow는 큐 없이 즉시 전송하고 응답 대기 (일별 처리 등 정합성 중요한 경로)
  • 상태 표시 사이드바 좌하단 점 + 상단 #ss-bar에 ok/saving/err 시각화
  • 설정값 Store.setSetting은 메모리·localStorage 폴백·서버 3중 저장
